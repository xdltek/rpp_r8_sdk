## 1. 系统要求

安装前请确认目标系统满足最低配置：

- **操作系统**：Ubuntu 20.04 LTS (x86_64)
- **系统内存**：≥ 16 GB DDR5
- **CPU 架构**：x86_64（支持 PCIe® Gen3）
- **编译器**：GCC 9.4.0 · **CMake**：≥ 3.26.5 · **Python**：3.11
- **构建工具**：`build-essential`、`linux-headers`

> 未满足要求可能导致安装失败或运行不稳定



## 2. 安装前：硬件验证

**目标**：确认 RPP 加速卡已在 PCIe 总线上正确枚举

执行命令：

```bash
lspci | grep Co-processor
```

**预期输出**：`XX:XX.X Co-processor: Device 1f2e:00a1 (rev 01)`

- `XX:XX.X` = BDF，例如 **05:00.0**
- 有 Co-processor 条目 → 识别成功

> 无设备时请检查插卡、PCIe 插槽及 BIOS 设置



## 3. SDK 安装 — 步骤 1：获取安装包

从官方渠道下载：

```
azurengine_sw_<version>_x86_Ubuntu.run
```

示例：`azurengine_sw_v1.7.2.4_x86_Ubuntu.run`



## 3. SDK 安装 — 步骤 2：卸载旧版本（推荐）

```bash
bash /usr/local/rpp/doc/uninstall.sh
```

- 移除所有 RPP 包、驱动及支持文件
- 若最终显示成功，可忽略轻微警告

> **重要**：请在干净系统或完全卸载后再安装，不支持多版本并存



## 3. SDK 安装 — 步骤 3：执行安装

```bash
sudo chmod +x azurengine_sw_v1.7.2.4_x86_Ubuntu.run
bash azurengine_sw_v1.7.2.4_x86_Ubuntu.run
```

安装过程中：

- 自动卸载并重装 RPP 驱动
- 检查并安装系统依赖

确认无报错并显示 SDK 版本与时间戳



## 4. 驱动安装与验证

**加载驱动**（每次重启后需重新执行）：

```bash
cd $HOME/azurengine/rpp_drv
sudo insmod rpp.ko
sudo chmod 666 /dev/rpp0_entire_ctrl /dev/ve0_entire_ctrl
```

**验证**：`dmesg` 无报错即表示加载成功

> 自动化配置请参考 XDL Demo 指南



## 5. 故障排除 — 驱动类

| 错误 | 处理 |
|------|------|
| `File exists` | 驱动已加载，无需操作 |
| `Invalid module format` | 重新编译：`make clean && make -j$(nproc)` 后重复加载步骤 |



## 5. 故障排除 — PCIe 未识别

**现象**：`RPP accelerator card not found on PCIe bus`

**可能原因**：插卡松动、插槽故障、BIOS PCIe 禁用、BIOS 过旧

**诊断**：

```bash
lspci | grep -i Co-Processor
sudo lspci -s "05:00.0" -vv | awk '/LnkCap:|LnkSta:/ {print}'
```

**处理**：重新插拔卡（30° 插入后压紧）、BIOS 启用 PCIe Gen4/5、关闭 ASPM、更新 BIOS



## 小结

**RPP SDK 安装流程**

1. 验证系统要求
2. 硬件识别：`lspci | grep Co-processor`
3. 下载 → 卸载旧版 → 执行安装
4. 加载驱动（每次重启后）
5. 遇问题时参考故障排除章节
